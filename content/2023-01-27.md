
CSSを一日中弄くり回していたら、iPhoneで見た場合とPCやAndroidで見た場合で挙動が異なる事に気が付きました・・・

競合した場合のCSSの適用順序が、どうやらwebkitの時だけ違うみたいで・・・

FirefoxとVivaldi(Chromiumベース)は同じ挙動だったのでどう考えてもApple様のせいですありがとうございました

webkitの変な挙動は以下を修正した時に発見しました

修正前
```
a {
	...
}

.foo a {
	...
}
```

修正後
```
a {
	...
}

h1.foo a {
	...
}
```
